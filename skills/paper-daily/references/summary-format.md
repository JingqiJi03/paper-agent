# Paper Summary Format

Shared by `paper-daily` and `paper-weekly`. For every important paper, summarize using this six-section structure. Detail level scales with the paper's relevance rank (see each skill's detail-level table).

## Paper: {title}

### 1. Challenge

Explain the core challenge the paper addresses.

Focus on:

- what problem the paper tries to solve
- why this problem is difficult
- why existing methods are insufficient
- where the main bottleneck lies (e.g., data, model architecture, training, inference efficiency, generalization, scalability, evaluation, or system design)

### 2. Motivation

Explain the authors' motivation.

Focus on:

- why they propose this direction
- what gap they identify in prior work
- why the problem matters for the field and for the user's research topic

### 3. Insight

Explain the key insight.

Focus on:

- the non-obvious idea behind the method
- what the authors explicitly model, condition on, optimize, or exploit
- why this idea could work
- what assumption the method depends on

### 4. Main method

Explain the method clearly and concretely.

Mention details when available:

- model / system architecture
- training objective and data
- inference procedure
- key algorithmic or mathematical design
- prompting, conditioning, or input/output strategy
- system pipeline and engineering choices
- benchmark / evaluation setup

### 5. Effect / Results

Explain the main results.

Mention details when available:

- benchmarks / datasets used
- headline quantitative metrics (accuracy, efficiency, speed, cost — whatever the field cares about)
- generalization / robustness results
- ablations that support the claims
- qualitative findings
- whether the evidence actually supports the paper's claims

### 6. Research Value

Judge the paper from a general research perspective, relative to the user's topic.

Discuss:

- reading priority: High / Medium / Low
- value as related work (worth citing?)
- value as a baseline (worth comparing against?)
- methodological inspiration
- benchmark or metric value
- trend importance
- whether it challenges current assumptions
- whether it should be read deeply later

Do not force a connection to the user's specific project if the paper does not support it.
